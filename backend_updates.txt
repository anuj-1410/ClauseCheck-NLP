### 1.1 Replace `en_core_web_trf` → `en_core_web_sm`

**Files Affected:** `entity_extractor.py`, `obligation_detector.py`, `responsibility_detector.py`

**Current State:**
- Using `en_core_web_trf` (spaCy Transformer Model with RoBERTa)
- Used for: tokenization, POS tagging, NER, dependency parsing

**Problem:**
- You are NOT using transformer embeddings or deep contextual NER features
- You are only using: dependency labels (`nsubj`, `dobj`), basic entity labels, token-level parsing
- You already use `sentence-transformers` for semantic understanding
- Running both = duplicate transformer stacks in memory (~unnecessary overhead)

**Action:**
- Replace `en_core_web_trf` with `en_core_web_sm` across all three files
- `en_core_web_sm` is fully sufficient for your actual usage (dependency labels + basic NER)

**Result:** Faster load times, lower RAM usage, no loss of functionality
